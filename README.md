# Autoscaling Strategy Comparison Project

**Project**: Comparative Analysis of Autoscaling Strategies: CPU Utilization vs. Request Rate

---

## 🚀 Artifact Appendix

This appendix provides all necessary details to understand, install, and evaluate the artifacts for this project, designed to meet academic reproducibility standards.

### 1. Artifact Information
*   **Repository URL:** [https://github.com/jaycee6666/autoscaling-strategy-compare](https://github.com/jaycee6666/autoscaling-strategy-compare)
*   **Key Artifacts:**
    *   **Infrastructure as Code (IaC):** Python scripts using the Boto3 SDK for automated AWS environment provisioning.
    *   **Automated Experiment Pipeline:** A master orchestration script for end-to-end comparative testing.
    *   **Data & Analysis:** Pre-run, stabilized datasets from rounds 13-22 and visualization scripts.

### 2. Portable Workflow Framework
To simplify evaluation, this project uses a standardized, script-based workflow that automates the entire environment setup. A `Dockerfile` is also provided for containerized execution.

**Prerequisites:**
*   Python 3.9+
*   AWS CLI, configured with an IAM user with `AdministratorAccess` permissions.

### 3. Step-by-Step Instructions

1.  **Clone Repository:**
    ```bash
    git clone https://github.com/jaycee6666/autoscaling-strategy-compare.git
    cd autoscaling-strategy-compare
    ```

2.  **Run Automated Setup:**
    ```bash
    # This script creates a virtual environment, installs dependencies,
    # and initializes project directories.
    python scripts/setup.py
    ```

3.  **Configure AWS Credentials:**
    Create and edit `config/.env` to include your AWS keys:
    ```
    AWS_ACCESS_KEY_ID=YOUR_KEY
    AWS_SECRET_ACCESS_KEY=YOUR_SECRET
    AWS_DEFAULT_REGION=us-east-1
    ```

4.  **Deploy and Run Experiments:**
    Execute the master pipeline script. This will provision the AWS stack, deploy the application, and run the full suite of experiments.
    ```bash
    python scripts/run_all_experiments.py
    ```
    *Note: The full pipeline runs for approximately 75 minutes. To simply verify the logic, you can run individual experiment scripts with a shorter duration, e.g., `python experiments/02_run_cpu_experiment.py --duration-seconds 60`.*

### 4. Evidence of Verification
The repository is structured to make verification straightforward.

*   **Runnable & Consistent:** All scripts are designed to be executed sequentially. The use of a dedicated load generator instance (`loadgen-host.json`) ensures that network conditions are consistent and not subject to local internet variability.
*   **Complete:** The `experiments/results_round13_22_median.json` file contains the final, aggregated data used in our report, which can be regenerated by running the pipeline. Raw data for each individual round is also available.
*   **Validation:** You can validate the health of the provisioned infrastructure at any time by running:
    ```bash
    python experiments/01_verify_infrastructure.py
    ```
